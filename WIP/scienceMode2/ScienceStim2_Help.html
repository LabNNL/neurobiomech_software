RehaStim2 device communication block

The stimulator interface block is used to control the stimulation device RehaStim2 in real-time. This block uses the ScienceMode2 communication protocol. The RehaStim2 device by HASOMED GmbH supports this protocol.

Connecting the Stimulator to the PC

The ScienceMode2 protocol expects the stimulator to be connected to a PC over a serial port style connection. The RehaStim2 stimulator device uses a built-in FTDI controller that manages this serial port style connection. Just connect the RehaStim2 via USB A/B connection cable to your PC.

Within a MS Windows operating system the USB port must be configured as a virtual communication port (VCP) using the correct driver. Please download this driver from Future Technology Devices International Ltd and select the FT232BM driver from the list associated with the correct platform.

Once the driver is installed the USB port can be set up and used as normal serial port.

Inputs

Current:

This input is a vector of values (e.g. [10 30 30 10]) representing the current amplitudes for each channel specified in the block parameters (Channels to be Stimulated). Thus if the specified channels are [1 2 5 8] then the vector dimension must be 4 in this case.

Pulsewidth:

This input is a vector of values (e.g. [200 300 100 300]) representing the pulse widths for each channel specified in the block parameters (Channels to be Stimulated).

Mode:

This input is a vector of values (e.g. [0 1 2 0]) representing the mode for each channel specified in the block parameters (Channels to be Stimulated).

Parameters

Sample Time:

The sample time of the block (specifies the update period for stimulation parameters; defines the main pulse interval for the One Shot Channel List Mode)

Serial Port:
  • sciencemode2 connects to /dev/sciencemode2 on Linux systems.
  • ttyUSB0 connects to /dev/ttyUSB0 on Linux systems.
  • ttyUSB1 connects to /dev/ttyUSB1 on Linux systems.
  • ttyUSB2 connects to /dev/ttyUSB2 on Linux systems.
  • ttyUSB3 connects to /dev/ttyUSB3 on Linux systems.
  • COM1 connects to COM1 on MS Windows systems.
  • COM2 connects to COM2 on MS Windows systems.
  • COM3 connects to COM3 on MS Windows systems.
  • COM4 connects to COM4 on MS Windows systems.
    …
  • COM8 connects to COM8 on windows systems.
Mode (if enabled):
  • 0 is for single stimulation pulses
  • 1 is for doublet stimulation pulses
  • 2 is for triplet stimulation pulses
Channels to be Stimulated:

A vector of channel numbers (1..8) to be used for stimulation (e.g. [1 2 5 8])

Main Pulse Interval:

The stimulation period (set this to 0 for One Shot Channel List Mode)

Inter Pulse Interval:

Defines the time between two pulse groups if doublets or triplets are used.

Low Frequency Channels:

A sub-set of the channels selected for low frequency stimulation
Low Frequency Factor:

Defines how many times the stimulation is skipped for the channels specified in low
frequency channels. (0 = no skip, 1 = skip once etc.)
